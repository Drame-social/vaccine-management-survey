# Data Dictionary: Synthetic Vaccine Management Survey Project

All datasets are synthetic and must not be interpreted as CDC, PEAR, VTrckS, provider, patient, or awardee data. The synthetic label appears in every row-level file.

## File names and table names

| File | Table name | Grain | Purpose |
|---|---|---|---|
| `core_survey_synthetic.csv` | `core_survey_synthetic` | 1 row per synthetic awardee | Core survey sections I, II, III, V, VI. |
| `vaccine_orders_wide_synthetic.csv` | `vaccine_orders_wide_synthetic` | 1 row per synthetic awardee | SAS-style wide ordered-dose table. |
| `vaccine_waste_wide_synthetic.csv` | `vaccine_waste_wide_synthetic` | 1 row per synthetic awardee | SAS-style wide wasted-dose table. |
| `vaccine_loss_reported_wide_synthetic.csv` | `vaccine_loss_reported_wide_synthetic` | 1 row per synthetic awardee | SAS-style wide reported vaccine-loss percentages. |
| `vaccine_incidents_synthetic.csv` | `vaccine_incidents_synthetic` | 1 row per synthetic awardee-incident category | Storage/handling incident categories. |
| `vaccine_order_waste_long_synthetic.csv` | `vaccine_order_waste_long_synthetic` | 1 row per synthetic awardee-vaccine type | Normalized analysis table for order, waste, and loss. |
| `awardee_vaccine_loss_summary.csv` | `awardee_vaccine_loss_summary` | 1 row per synthetic awardee | Processed awardee summary. |
| `vaccine_type_loss_summary.csv` | `vaccine_type_loss_summary` | 1 row per vaccine type | Processed vaccine-type summary. |
| `staff_training_gap_summary.csv` | `staff_training_gap_summary` | 1 row per staffing metric | Processed staff training summary. |

## Cross-table validation rules

- `core_survey_synthetic.awardee_id` is the primary key.
- All vaccine order/waste and incident rows must map to a valid `awardee_id`.
- `doses_wasted <= doses_ordered` for every awardee-vaccine row.
- `calculated_loss_pct = 100 * doses_wasted / doses_ordered` where `doses_ordered > 0`.
- Section IV total waste should reconcile to Section V incident-dose totals; intentional synthetic discrepancies are flagged in `awardee_vaccine_loss_summary.reconciliation_flag`.
- Every row-level file must contain `synthetic_data_notice`.
